Run the fixed local benchmark — the same seed user, the same questions, and a fixed tool set every run — so you can watch the agent get scored and compare runs. (The hosted validator rotates a fresh dataset; this one is reproducible.)

How to read this

  • Composite = 0.6 × tool score + 0.4 × memory score (the headline number).
  • Tool score (per case): ½ tool-accuracy + ½ response-quality judge — tool-accuracy = matched ÷ expected tools (−0.1/extra; no-tool case = 1.0 if nothing called); the LLM judge rates the reply's helpfulness + accuracy. Mirrors the backend's DittoBench tool composite.
  • Memory score (per question): an LLM judge (yes/no), the same LongMemEval QA judge the published scores used — not a substring match.
  • Score colors: 1.0 · partial · 0.
  • Inputs are fixed, but the model is stochastic — expect small run-to-run variation.
